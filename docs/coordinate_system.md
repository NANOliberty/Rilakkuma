# 좌표계 규약 (Coordinate System)

> 두 팀원이 같은 좌표/스케일로 작업해 통합 시 비례가 깨지지 않게 하는 기준 문서.
> 원본은 PROJECT_SPEC.md 5.2 절. 여기서는 구현 관점으로 정리한다.

## 축 방향

```
        +Y (위)
         |
         |
         +-------- +X (캐릭터의 오른쪽 = 보는 사람의 왼쪽)
        /
       +Z (캐릭터가 보는 방향 = 화면 앞쪽)
```

- 카메라 초기 위치: 정면 살짝 위(+Z 쪽, phi≈15°).
- 모든 메쉬는 **원점 기준**으로 그리고 SceneNode 변환으로 배치.

## 기준 스케일

- **머리 반지름 = 1.0** ← 모든 크기의 기준.

### 로컬 좌표 환산 팁
머리 자식들은 머리 노드(0,1.5,0)의 **로컬 좌표**로 둔다 = (월드 - (0,1.5,0)).
예) 왼쪽 눈 월드 (-0.45,1.7,0.85) → 머리 로컬 (-0.45, 0.2, 0.85).
(Head.cpp 에 이미 이렇게 적용돼 있음.)

## 인사 포즈 회전 (PROJECT_SPEC 5.2.2)

| 부위 | 축 | 각도 |
|---|---|---|
| 왼팔(번쩍) | Z | +130° (+ X -15°) |
| 오른팔(배쪽) | Z | -25° (+ X +25°) |
| 왼다리 | Z | -10° |
| 오른다리 | Z | +10° |
| 머리(선택) | Y | 0 ~ -10° |

> 회전 중심 = 어깨/고관절(부품 위쪽 끝). 두 축 합성은 SceneNode 중첩으로.
